    Paper Prototype

    Lessons Learned

    • Faster
    • Easily Modified
    • Cheap! Cost you almost nothing

    Fail Fast, Iterate Faster

    Lessons Learned

    • Once you got the idea, do it fast
    • No Pretty GUI required during this process
    • Make more concept, when it fail, leave it
    • Move to next prototype

    Continue, not Build New

    Lessons Learned

    • More than 50% success games are franchise
    • Maintain to engange more user
    • There's always room for improvement
    • if not franchise, it's a sequel

    Test! Need More Test!

    Lessons Learned

    • Although painful, it's required
    • User experience is one of the hardest
    • UX is the hardest to get the right feels
    • The question is: ‘What’s the hardest thing on development’. It’s testing, well, we always need more tester that can give us feedback

    Random Number Generation

    Math for Game Programmers - Random Numbers

    • Just "random"
      • return 0 to MAX_INTEGER
    • Pseudo-Random
      • provide variety
      • provide reproducibility
        • using same seed
        • seed is fragile, be careful

    Randomness affect Player’s Psychology

    • Create Disbelief
      • 70% means yes, I think there’s a chance to get it
      • 99% means guaranteed
      • 0.1% means almost never

        but, with LARGE scale players, they will hit this, a lot

        They will feel lucky (or unlucky)

    Reloading State & Random

    • Re-roll in encounter
        Allow state manipulation
    • Pre-roll (on initialize)
        Allow future prediction

    Types of Randomness

    • Independent randomness
        Previous roll does not affect the next one
        Example: dice
    • Dependent randomness
        Each random has the chance to alter the probabilities
        Example: card deck
